The Exp and Expm meta-models

  1. Define a model of Expressions (Exp) based on the 'traditional' (infix) syntax
  2. Define a (more handy) model of Expressions (Expm) as binary tree (prefix concrete syntax)
  3. Build an expression evalutator in Java (EvalExpm) and in Xtend (EvalExpmXtend) by using the binary expression model produced by Xtext
  4. Define a Test unit (TestExpm) to show how evalutate an expression given a String in prefix form
  5. Evaluate an expression including sensors (EvalSensor)
  6. Build a ModelToModel (M2M) (ExpToExpm) mapping in order to transform a binary expression of type exp.E into an expression of type expm.Expression.
  7. Build an utility class in Java (TestEvalWithM2M) that allows to evaluate expressions espressed as String in infix form.
  8. Expressions as distributed systems: : build a software system able to execute sensor expressions written in Exp as distributed systems in which each operstor can be allocated and work on a different computational node.
